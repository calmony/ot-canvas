# Contributing

This is a prototype. Contributions welcome but not expected.

## What's helpful

- **Bug reports** — anything that renders wrong on your browser/OS combo
- **Vendor data corrections** — wrong model numbers, MAC OUIs, EOL dates, OS platforms
- **New asset categories** — protection relays, drives, robots, motion controllers, safety PLCs
- **Real CVE counts** — if you want to wire in actual NVD data
- **Additional Purdue scenarios** — what would a refinery, water treatment, or power gen plant look like?

## What's out of scope

- Backend integrations (this is intentionally frontend-only)
- Actual scan parsing (the import flow is a UI demo)
- Customer-specific or government-network data

## Code style

Plain HTML/CSS/JS. No build step, no framework, no preprocessor. Keep it simple enough that a controls engineer with basic web knowledge can read and modify it.

## Pull requests

Fork, branch, PR. Keep changes focused. Screenshots help.
